`src/nexuscore/utils/tree_sitter_checker.py` の速度最適化と安定性向上を実装しました。
特に「大きめリポジトリを解析しても体感で待てる速度」に近づけることを目標としました。

## 実装ステップ

### 1. 現状コードのリーディングとTODOコメント追加

**変更内容**:
- エントリポイント関数を特定: `analyze_file()`, `analyze_project()`, `analyze_source_code()`

  - ファイルリスト取得 (`analyze_project`)
  - 並列処理のオーバーヘッド (`analyze_project`)

### 2. 簡易プロファイリングフックの追加

**実装内容**:
- `_profiling_stats` 辞書で統計を収集:
  - `total_files`: 解析対象ファイル数
  - `cache_hits`: キャッシュヒット数
  - `cache_misses`: キャッシュミス数
  - `total_time`: 総解析時間
  - `file_times`: ファイルごとの解析時間リスト
- 環境変数 `NEXUS_TREESITTER_ENABLE_PROFILING` で有効/無効を制御（デフォルト: 無効）
- プロファイリング有効時は解析完了後に統計をログ出力

**効果**: 解析時間のボトルネックを特定し、さらなる最適化の指針を得られる

### 3. キャッシュの導入（最優先）

**実装内容**:
- ファイルパス × ファイル内容ハッシュ（SHA256）をキーに解析結果をキャッシュ
- `_cache` 辞書でメモリキャッシュを管理
- `_compute_content_hash()`: ファイル内容のハッシュを計算
- `_get_cache_key()`: キャッシュキーを生成
- `analyze_source_code()` でキャッシュチェックと保存を実装
- 環境変数 `NEXUS_TREESITTER_ENABLE_CACHE` で有効/無効を制御（デフォルト: 有効）
- `clear_cache()`: キャッシュをクリアするメソッドを追加

**効果**: 同じファイルを複数回解析する場合、2回目以降はキャッシュから取得されるため大幅に高速化

### 4. 並列処理の改善（既存のThreadPoolExecutorを活用）

**実装内容**:
- 既存の `ThreadPoolExecutor` を活用（I/O バウンドのため Thread を選択）
- 環境変数 `NEXUS_TREESITTER_MAX_WORKERS` で並列度を制御可能（デフォルト: CPU コア数）
- `CONFIG` 辞書に環境変数から設定を読み込む機能を追加

**効果**: 大量ファイルの解析時に並列処理により処理時間を短縮

### 5. タイムアウトとフェイルセーフの強化

**実装内容**:
- `FutureTimeoutError` をインポートしてタイムアウト処理を実装
- `analyze_project()` で `future.result(timeout=CONFIG['timeout_seconds'])` を使用
- タイムアウト発生時は該当ファイルのみ失敗として記録し、処理全体は継続
- 環境変数 `NEXUS_TREESITTER_TIMEOUT_SEC` でタイムアウト時間を制御可能（デフォルト: 60秒）
- エラー発生時もログに警告を残し、処理全体は継続

**効果**: 問題のあるファイルが混ざっていても処理全体が落ちずに継続できる

### 制限事項
1. **プロセス内キャッシュ**: キャッシュはプロセス内のみ有効（永続化なし）
2. **メモリ使用量**: 大量ファイルを解析する場合、キャッシュがメモリを消費する可能性がある

### 移行時の注意点
