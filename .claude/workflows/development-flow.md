# NexusCore 開発ワークフロー（軽量版）

このドキュメントは、NexusCore プロジェクトでの開発フローを定義します。

---

## Tier 2 タスク（通常開発）のフロー

**対象**: ビジネスロジック実装、UI実装、バグ修正、リファクタリング、テスト追加

```
ユーザー指示
    ↓
直接実装開始
    ↓
完了
```

**例**:
```
ユーザー: CoderAgentに機能追加して
Claude Code: [直接実装開始]
```

**例**:
```
ユーザー: このバグを直して
Claude Code: [直接実装開始]
```

---

## Tier 1 タスク（重要）のフロー

**対象**: 認証・認可、決済処理、データ移行、公開API設計、セキュリティ関連

```
ユーザー指示
    ↓
Tier Check で判定
    ↓
Tier 1 と判定
    ↓
簡易仕様確認（3項目）
  ├─ 何を作るか (1-2文)
  ├─ 主要な制約
  └─ 影響範囲
    ↓
ユーザー確認
    ↓
実装開始
    ↓
重要な決定が必要な場合
    ↓
Decision Recorder で記録
    ↓
実装完了
```

**例**:
```
ユーザー: JWT認証機能を実装して

Claude Code:
[Tier Check]
このタスクは Tier 1（認証・認可）です。
簡易仕様を確認させてください:

1. 何を作るか: JWT 方式のユーザー認証機能
2. 主要な制約:
   - アクセストークン有効期限: 15分?
   - リフレッシュトークン: あり/なし?
   - パスワードハッシュ: bcrypt?
3. 影響範囲: 新規ファイル作成 or 既存ファイル修正?

ユーザー: 15分、リフレッシュあり、bcrypt、新規でOK

Claude Code:
## 簡易仕様

### 何を作るか
JWT 方式のユーザー認証機能

### 主要な制約
- アクセストークン有効期限: 15分
- リフレッシュトークン: あり
- パスワードハッシュ: bcrypt

### 影響範囲
- auth.py (新規作成)
- middleware.py (新規作成)
- user_model.py (token_version カラム追加)

上記仕様で実装を開始します。よろしいでしょうか？

ユーザー: はい

Claude Code: [実装開始]
```

## 週次コードレビューフロー

```
ユーザー「今週レビューして」
    ↓
Code Reviewer Agent 起動
    ↓
並列で複数の重要ファイルを分析
  ├─ file1.py (Code Reviewer subagent)
  ├─ file2.py (Code Reviewer subagent)
  └─ file3.py (Code Reviewer subagent)
    ↓
サイドバーで進捗監視
    ↓
レビュー結果を表示
    ↓
重大な指摘 → 即時修正
軽微な指摘 → Issue化
決定記録 → Decision Recorder で記録
```

**自動化**: Code Reviewer Agent が週次で自動実行

---

## エージェント間連携

### Tier 1 タスク実装時の連携

```
Tier Check
    ↓ (簡易仕様確認)
実装開始
    ↓ (重要な決定が発生)
Decision Recorder
    ↓ (実装完了)
完了
```

### 週次レビュー時の連携

```
Code Reviewer
    ├─ 重大な指摘 → 即時修正
    ├─ 軽微な指摘 → Issue化
    └─ 決定記録 → Decision Recorder で記録
```

## 開発速度の目安

## ルール適用範囲

### 適用するルール（最小限）

- ✅ プロジェクトタグ必須（`#project: NexusCore`）
- ✅ API アーキテクチャ（FastAPI, /api/v1）
- ✅ 認証方式の一貫性
- ✅ エラーコードカタログ
- ✅ Tier 分類と開発フロー

### 廃止した規約（重い）

- ❌ 2セクション構成（人間向け/Cursor向け分離）
- ❌ CR完了レポートの7セクション構成
- ❌ 小さな修正でのSpec必須化
- ❌ unified diffのみ制約
- ❌ SDK生成・TestPyPI公開手順の規約
