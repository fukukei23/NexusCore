---
name: tier-check
description: タスクが Tier 1（重要）か Tier 2（通常）かを判定する

## 目的
ユーザーからのタスク依頼を、適切な開発フローに振り分けます。

## Tier 1 タスク（重要）

**定義**: 以下のカテゴリに属するタスク

- **認証・認可**: JWT、OAuth、ロール管理、権限チェック
- **決済処理**: クレジットカード、サブスクリプション、課金
- **データ移行**: データベーススキーマ変更、データマイグレーション
- **公開API設計**: 外部公開する API の新規追加・変更
- **セキュリティ関連**: 暗号化、トークン管理、脆弱性修正

**判定例**:
```
ユーザー: JWT認証機能を実装して
判定: Tier 1（認証・認可）

ユーザー: 決済エンドポイントを作って
判定: Tier 1（決済処理）

ユーザー: /api/v1/users を追加して
判定: Tier 1（公開API設計）
```

## Tier 2 タスク（通常）

**定義**: 以下のカテゴリに属するタスク

- **ビジネスロジック実装**: 機能追加、エージェント開発
- **UI実装**: Gradio コンポーネント、画面追加
- **バグ修正**: 既存機能の不具合修正
- **リファクタリング**: コード整理、パフォーマンス改善
- **テスト追加**: テストカバレッジ向上

**判定例**:
```
ユーザー: CoderAgentに機能追加して
判定: Tier 2（ビジネスロジック実装）

ユーザー: ログイン画面を作って
判定: Tier 2（UI実装）

ユーザー: このバグを直して
判定: Tier 2（バグ修正）

ユーザー: テストを追加して
判定: Tier 2（テスト追加）
```

## 曖昧な場合の処理

**ルール**: 判定が難しい場合は安全側に振る

```
ユーザー: ユーザー機能を作って
判定: Tier 1（公開APIの可能性があるため）
```

**質問**: 不確定な点があれば簡潔に確認

```
ユーザー: 機能Xを作って
質問: これは公開APIとして提供するものでしょうか？
- はい → Tier 1
- いいえ → Tier 2
```

## 出力形式

```
## Tier 判定結果

### タスク
[タスクの説明]

### Tier
Tier 1 または Tier 2

### 理由
[判定理由]

### 推奨フロー
[実装フローの説明]
```
