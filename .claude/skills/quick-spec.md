## 目的
Tier 1 タスク（重要）の実装前に、最低限必要な情報を確認します。

## 確認項目

### 1. 何を作るか（1-2文）

タスクの目的と作成するものを簡潔に説明

**例**:
- JWT 方式のユーザー認証機能
- 決済プロセスの API エンドポイント

### 2. 主要な制約

実装時の重要な制約条件を箇条書きで

**例**:
- アクセストークン有効期限: 15分
- リフレッシュトークン: あり
- パスワードハッシュ: bcrypt
- 決済プロバイダ: Stripe

### 3. 影響範囲

作成・変更するファイルやモジュール

**例**:
- auth.py (新規作成)
- middleware.py (新規作成)
- user_model.py (token_version カラム追加)
- src/nexuscore/api/routes/auth.py (新規作成)

## 使用手順

1. **タスク分析**: ユーザーからの指示を3項目に整理
2. **確認提示**: ユーザーに仕様を提示して確認を取る
3. **実装開始**: 確認後、実装を開始

## 出力形式

```
## 簡易仕様確認

### 何を作るか
[1-2文の説明]

### 主要な制約
- 制約1
- 制約2
- 制約3

### 影響範囲
- ファイル1 (新規/変更)
- ファイル2 (新規/変更)
- ファイル3 (新規/変更)

上記仕様で実装を開始します。よろしいでしょうか？
```

## 注意点

- 簡潔さを重視: 1-2文で済ませる
- 具体的な制約のみ: 実装に直接影響するもの
- 不確定な点は事前に質問: 決定が必要な箇所は確認

## 非対象

Tier 2 タスク（通常）ではこのスキルを使用しない

- バグ修正
- 小さな機能追加
- リファクタリング
- テスト追加

これらは直接実装開始
