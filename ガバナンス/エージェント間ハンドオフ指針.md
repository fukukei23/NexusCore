# エージェント間ハンドオフ指針

> 元資料: claude-config「セッションハンドオフ自動生成」（2026-05-05）
> 対象: NexusCore 14自律エージェント

## 目的

NexusCoreの14自律エージェント間でコンテキストを引き継ぐためのフォーマットと設計指針を定義する。

## 既存の関連実装

| 機能 | 場所 | 役割 |
|---|---|---|
| セッションステート | `docs/archive/SESSION_HANDOFF.md` | 手動ハンドオフのテンプレート（アーカイブ） |
| リトライコンテキスト | `agents/` 各ファイル `RetryContext` | 単一エージェント内のエラー復旧 |
| LLM呼び出しログ | `logs/llm_calls.jsonl` | 全呼び出しのJSONL記録 |
| ジョブ状態機械 | `core/` | ジョブ全体の進捗管理 |

## ハンドオフフォーマット

エージェント間で引き継ぐべき情報:

```json
{
  "source_agent": "coder_agent",
  "target_agent": "debugger_agent",
  "task_id": "TASK-001",
  "timestamp": "2026-05-13T12:00:00Z",
  "status": "completed_with_issues",
  "artifacts": {
    "files_modified": ["src/module.py"],
    "test_results": {"passed": 8, "failed": 1},
    "llm_calls": 12,
    "total_cost_usd": 0.03
  },
  "context": {
    "what_was_done": "リファクタリング実施",
    "what_failed": "test_edge_case が失敗",
    "next_steps": ["エッジケースのデバッグ", "テスト修正"]
  }
}
```

## 設計指針

### 1. 軽量ステート vs 完全セッション

| パターン | 用途 | 保持期間 |
|---|---|---|
| `RetryContext` | 単一タスク内のリトライ | タスク実行中のみ |
| ジョブステート | ジョブ全体の進捗 | ジョブ完了まで |
| ハンドオフ | エージェント間のコンテキスト転送 | 受信エージェントのタスク完了まで |

### 2. ログベースの進捗追跡

- `llm_calls.jsonl` に `task_id` フィールドを追加することで、タスクごとのLLM使用量を追跡可能
- 各エージェントの実行ログをJSONLで統一フォーマット化
- セッション跨ぎの場合は、ログファイルから前回のコンテキストを復元

### 3. コールドスタート対策

claude-configの「セッションハンドオフ自動生成」パターン:

1. **Stop時**: セッション終了時にハンドオフ情報（未解決問題・git状態・次ステップ）をファイルに保存
2. **Start時**: 次セッション開始時にハンドオフファイルを読み込み、前回の文脈を復元
3. **フォーマット**: Markdown（人間・AI両方可読）

NexusCoreでは、オーケストレーター（`core/`）がこの役割を担う。各エージェントの終了時にハンドオフJSONを生成し、次のエージェント起動時に読み込む設計とする。

### 4. 実装の優先順位

1. **Phase A**: `llm_calls.jsonl` に `task_id` を追加（コード変更最小）
2. **Phase B**: 各エージェントに `_generate_handoff()` メソッドを追加
3. **Phase C**: オーケストレーターがハンドオフJSONを管理・転送
