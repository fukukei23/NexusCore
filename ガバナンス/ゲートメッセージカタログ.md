# Gate Message Catalog

> **Gate 判定時のメッセージテンプレート集**

本カタログは、Gate 判定時に使用する標準メッセージテンプレートを提供します。

---

## Fast Gate メッセージ

### Fast Gate 適用通知

```
この変更は Fast Gate で処理されます。
- IRG: 不要
- Spec 更新: 不要
- 最速で適用可能
```

### Fast Gate 適用範囲外

```
この変更は Fast Gate の適用範囲外です。
変更内容を確認し、適切な Gate（Standard / Strict）を適用してください。
詳細: docs/ARCHITECTURE.md セクション 5
```

---

## Standard Gate メッセージ

### Standard Gate 適用通知

```
この変更は Standard Gate で処理されます。
- IRG: 条件付き（変更内容により判断）
- Spec 更新: 必須
- Test 更新: 必須
```

### Standard Gate 要件確認

```
Standard Gate の要件を確認してください：
1. Spec が存在し、更新されているか
2. Test 要件が Spec に明示されているか
3. 実装が Spec と Test を満たしているか
```

---

## Strict Gate メッセージ

### Strict Gate 適用通知

```
この変更は Strict Gate で処理されます。
- IRG: 必須
- Spec 更新: 必須
- Test 更新: 必須
- Decision Log: 必須

実装前に以下を確認してください：
1. Spec が存在し、最終挙動が定義されているか
2. 無限ループ / 再試行上限が定義されているか
3. 実装詳細（具体値・コード）が混入していないか
```

### Strict Gate 必須フロー

```
Strict Gate では、以下の順序を厳守してください：
1. Spec 確認（実装前）
2. IRG（Independent Review Gate）
3. Spec → Test → Implementation の順序固定

IRG の Verdict が Approve でなければ実装禁止です。
```

### Strict Gate 適用範囲

```
以下の変更は Strict Gate が適用されます：
- エラー処理・リトライ・自律挙動
- 外部 API 連携
- コスト・レート制限
- セキュリティ関連

CR-NEXUS-051 クラス（エラー処理・リトライ・自律修復）は常に Strict Gate です。
```

---

## IRG（Independent Review Gate）メッセージ

### IRG 必須通知

```
この変更には IRG（Independent Review Gate）が必須です。

IRG のルール：
- IRG は「実装に進んでよい Spec か」だけを判定する
- 実装案・コード・修正案を提示してはならない
- 出力は Verdict（Approve / Reject）と指摘事項のみ

IRG の Verdict が Approve でなければ実装禁止です。
```

### IRG Verdict 通知

```
IRG Verdict: [Approve / Reject / Conditional]

[指摘事項があれば記載]
- High Severity: [内容]
- Medium Severity: [内容]
- Low Severity: [内容]
```

## Gate 判定エラーメッセージ

### Gate 未判定

```
Gate が判定されていません。
変更内容を確認し、適切な Gate（Fast / Standard / Strict）を適用してください。
詳細: docs/ARCHITECTURE.md セクション 5
```

### Gate 判定矛盾

```
Gate 判定に矛盾があります。
変更内容と適用 Gate が一致していません。
詳細: docs/ARCHITECTURE.md セクション 5.2 Change-Type Based Gate Matrix
```

## 実装プロトコル違反メッセージ

### Spec 未確認

```
実装前に Spec を確認してください。
Strict Gate では、Spec の存在と完全性が必須です。
```

### IRG 未実施

```
IRG（Independent Review Gate）が未実施です。
Strict Gate では IRG が必須です。
IRG の Verdict が Approve でなければ実装禁止です。
```

### 順序違反

```
実装順序が正しくありません。
Strict Gate では、Spec → Test → Implementation の順序を厳守してください。
```
