# 衛生チェック手順（限定対象版）

> **重要**: 衛生チェック対象は **ガバナンス/**, **プロジェクトプロファイル/**, **判断ログ/** に限定します。`docs/` 配下は対象外です。

## 目的

STIT+IRG ガバナンス資産（ガバナンス/**, プロジェクトプロファイル/**, 判断ログ/**）に絶対パスや環境依存パスが混入していないことを検証します。

## 対象範囲

**チェック対象（限定）:**
- `ガバナンス/**/*.md`
- `プロジェクトプロファイル/**/*.md`
- `判断ログ/**/*.md`

**チェック対象外:**
- `docs/**` - 既存の運用手順（WSLパス等）を含むため、対象外
- その他の既存ドキュメント

## 実行コマンド

```bash
# 限定対象の衛生チェック
cd /home/yn441611/NexusCore

# 禁止パターン検出（No output = PASS）
for f in ガバナンス/**/*.md プロジェクトプロファイル/**/*.md 判断ログ/**/*.md; do
  if [ -f "$f" ]; then
    grep -nE "(C:|D:|/home/|/Users/|\\\\wsl|\\\\wsl\.localhost|/mnt/|/tmp/[^/]+\.md|/var/|/etc/)" "$f" 2>/dev/null
  fi
done
```

**期待結果**: 出力なし（No output = PASS）

## 注意事項

- `docs/` 配下の既存ドキュメントは、WSL環境での運用手順として絶対パスを含む場合があります。これらは対象外です。
- 衛生チェックは **新規作成・変更したガバナンス資産のみ** を対象とします。
- SSOTはGitツリーのみです。`git status`, `git diff`, `git ls-files` を根拠にします。

## 再発防止

- 全.mdファイルを対象にした衛生チェックは禁止
- `docs/` 配下の既存ドキュメントを自動置換しない
- 衛生チェック対象は明示的に限定する

## 破壊的コマンドガード

> 元資料: claude-config「破壊的コマンドガード」（2026-05-05）

NexusCore CLI（`main_cli.py`）やエージェントが生成するシェルコマンドに対して、以下の危険パターンを検知・ブロックすること。

### ブロック対象パターン（19種）

| カテゴリ | パターン | 例 |
|---|---|---|
| ファイル削除 | `rm -rf /`, `rm --no-preserve-root`, `sudo rm -rf /` | システム全体削除 |
| ディスク破壊 | `dd if=... of=/dev/`, `mkfs.`, `> /dev/sd` | デバイス直接書き込み |
| 権限変更 | `chmod -R 777 /`, `chown -R ... /` | 全ファイル権限放开 |
| Git強制操作 | `git push --force origin main`, `git reset --hard` | 履歴破壊・強制上書き |
| SQL破壊 | `DROP DATABASE`, `TRUNCATE TABLE`, `DELETE FROM`（WHEREなし） | データベース全削除 |
| リモート実行 | `curl ... \| sh`, `wget ... \| sh` | 未検証スクリプト実行 |
| Docker | `docker system prune --all`, `docker rm -f`（稼働中コンテナ） | コンテナ・イメージ全削除 |

### 設計指針

- `guard/policy_engine.py` の `SecurityInput(secret_found=False)` を拡張し、コマンドパターン検知を組み込むことを検討
- エージェントが生成するコマンドは実行前にパターンマッチで検証
- ブロック時は `GuardDecision.BLOCK` + 理由コード `DESTRUCTIVE_COMMAND` を返却
- 正規表現による grep -qiE 照合が実績あり（19パターン全検証済み）

### 実装例（参考）

```bash
# 危険パターンの正規表現（grep -qiE で照合）
PATTERNS=(
  'rm\s+(-[a-zA-Z]*f[a-zA-Z]*\s+|-rf\s+|--no-preserve-root\s+)?/(\s|$)'
  'dd\s+if=.*\s+of=/dev/'
  'chmod\s+-R\s+777\s+/'
  'git\s+push\s+(-f|--force)\s+.*\s+(main|master)'
  'DROP\s+DATABASE'
  'curl.*\|\s*(ba)?sh'
  'docker\s+system\s+prune\s+.*--all'
)
```

## pytest 実行方法

### 通常のテスト実行

```bash
# 全テストスイートを実行
pytest -q

# 特定のテストスイートを実行
pytest -q tests/trace
pytest -q tests/guard
```

### Mutation Testing 実行

```bash
# mutation testing 用の限定収集（tests/agents のみ）
NEXUS_MUTATION_TEST=1 pytest -q tests/agents
```

**注意**: `NEXUS_MUTATION_TEST=1` が設定されている場合、`tests/conftest.py` と `tests/agents/conftest.py` が `tests/agents/` 以外のテストディレクトリを無視します。通常の開発時はこの環境変数を設定しないでください。
