# Fast Gate Quickstart

> **Fast Gate 適用範囲**: ドキュメント修正・文言変更・軽微な内部改善

Fast Gate は、**最速で変更を適用するための Gate** です。  
通常の利用や軽微な変更は Fast Gate のままで問題ありません。

---

## Fast Gate の特徴

- **IRG 不要**: Independent Review Gate は不要
- **Spec 更新不要**: 仕様書の更新は不要
- **最速適用**: 最小限の手順で変更を適用可能

---

## Fast Gate 適用例

以下の変更は Fast Gate で処理できます：

- README の更新
- コメントの修正
- エラーメッセージの文言変更
- ログ出力の改善
- 軽微なリファクタリング
- パフォーマンス最適化（既存機能の改善）

---

## Fast Gate での作業手順

1. **変更を実施**
   - ドキュメント修正・文言変更・軽微な改善を実施

2. **テスト確認（必要に応じて）**
   - 既存テストが通ることを確認
   - 新規テストは不要（軽微な変更の場合）

3. **コミット**
   - 変更をコミット

4. **完了**
   - IRG や Spec 更新は不要

---

## 注意事項

⚠️ **以下の変更は Fast Gate では処理できません**：

- エラー処理・リトライ・自律挙動の変更 → **Strict Gate**
- 外部 API 連携の変更 → **Strict Gate**
- コスト・レート制限の変更 → **Strict Gate**
- セキュリティ関連の変更 → **Strict Gate**
- 新機能追加 → **Standard Gate**
- API 変更 → **Standard Gate**

詳細は `docs/ARCHITECTURE.md` セクション 5「Governance: Change Classification and Gate Policy」を参照してください。
