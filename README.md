# NexusOS: The Operating System for Autonomous Innovation

## 📜 Vision

NexusOS is not an application; it is a complete, self-governing Operating System designed to autonomously create, manage, and scale digital enterprises.

This OS provides a secure, resource-aware, and process-driven Kernel, upon which autonomous AI agents (System Services) run to perform complex tasks like software development, business strategy, and even self-governance. The entire OS is designed to be portable and can be deployed with a single command, embodying the concept of an "AI Organization-on-a-Chip."



## ✨ Key Features

- **Autonomous Development Loop**: An interactive Gradio UI allows for human-AI collaboration. The system can take requests, generate code, create its own tests, and automatically run a self-healing loop to fix errors.
- **Enterprise-Grade Governance**: All operations are pre-validated against budget and security policies by the **Nexus Protocol Engine (NPE)**. Every code change is automatically committed to Git, ensuring a complete audit trail.
- **Human-in-the-Loop Governance**: Key decisions, such as changes to the system's own development policies, are proposed by AI but require final approval from a human administrator via a dedicated web interface.
- **AI Peer Review**: To ensure quality, code generated by one AI agent is automatically reviewed and validated by a different AI model before being accepted, simulating a human peer-review process.
- **Advanced Code Analysis**: The system uses Tree-sitter for syntactic analysis and a suite of quality tools (`pylint`, `mypy`, `bandit`) to ensure all generated code meets the highest standards.

## 🏗️ Architecture Overview

NexusOS is built on a layered architecture, just like a traditional operating system.



-   **Layer 0: The Kernel (`nexus_os_kernel.py`)**: The trusted core of the OS. It manages security (NPE), process integrity (VCS, AI Peer Review), and resources (Sandbox).
-   **Layer 1: System Services (Agents)**: The AI Agent workforce (`CEO`, `Coder`, `Guardian`, `Constitutional Council`, etc.) that run on top of the Kernel and perform the core functions of the organization.
-   **Layer 2: The Shell (Interfaces)**: The Gradio Web UI (`gradio_app/`), Flask API Server (`api/server.py`), and voice interfaces that allow humans and other systems to interact with the OS.
-   **Layer 3: Applications**: High-level applications that run on the OS, such as the **Venture Capitalist Agent (`ventures/vc_agent.py`)**, which uses the OS's core capabilities to launch new, independent instances of NexusOS.

## 🚀 Quick Start

**Prerequisites**: Git, Python 3.11+, and Docker.

1.  **Configure Environment**: Copy `.env.template` to `.env` and fill in the required API keys.
2.  **Install Dependencies**:
    ```bash
    pip install -r requirements.txt
    ```
3.  **Launch the System**: Use the provided PowerShell script to launch all services.
    ```powershell
    .\launch_all.ps1
    ```
4.  **Access**:
    -   **Gradio UI**: `http://localhost:7860`
    -   **Flask API**: `http://localhost:5000`

## 🗺️ Project Structure

-   **`/src/nexuscore`**: The core source code for the OS Kernel and all AI Agents.
-   **`/app`**: A sample Flask application managed by NexusOS.
-   **`/config`**: System-wide configuration files, including `policy_rules.json`.
-   **`/tests`**: Unit and integration tests for the NexusOS components.
-   **`/logs`**: Contains all audit and operational logs.
-   **`/sandbox_repo`**: The Git repository where AI-generated code is developed and versioned.
